发表于 2019-02-01 | 分类于 vue

####前提
我所做系统是基于vue+view-design开发的，对于ie浏览器的兼容性比较差，所以需要做些额外的配置兼容下ie浏览器。

上面的配置是仅对源码(src/)有效，对依赖包无效。
对于配置说明，可见：https://github.com/browserslist/browserslist

一个默认的VueCLI项目会使用 @vue/babel-preset-app，它通过 @babel/preset-env 和 browserslist 配置来决定项目需要的 polyfill。

useBuiltIns 有三个设置选项：

  • false：不做任何操作
  • entry：根据浏览器版本的支持，将 polyfill 需求拆分引入，仅引入有浏览器不支持的polyfill
  • usage：检测代码中 ES6/7/8 等的使用情况，仅仅加载代码中用到的 polyfill
